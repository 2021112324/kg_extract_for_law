# 问题
1. 专业领域强上下文关联复杂文本，通过本体大模型抽取知识图谱效果极难提升
   - 不设计专门流程，仅通过设计本体和改进图谱抽取方案的方式难以抽取出高准确度的知识图谱
   - 要提升知识图谱质量，一是设计图谱质检和补全，二是设计专门流程

# 带条款的法规文件
1. 通过正则匹配切分文件为法规文件基础信息（含第一条）和条款
   - 输入的图谱标签+文件名称+类型作为实体id前缀
   - 根据“第X条”分割条款，并将“第X章”、“第X节”加入条款属性
   - 将切分结果保存至json文件中
2. 对切分文件进行抽取
   - 对基础信息进行抽取，获取法规文件基础信息
   - 对条款进行抽取，抽取条款分点、依据的条款
   - 设置缓存机制，避免重复抽取

# 案例
1. 确定下来哪些内容需要，哪些内容不需要
   - 裁判文书网爬取文件分类粒度多少？（一审、二审、再审；刑事、民事、行政；判决书、裁定书、决定书；制裁决定书、管辖裁定书、上诉管辖裁定书）
   - 例如：对于一审判决书，结构例如：
   ```markdown
   一审
   【一】首部（立案与审理状态）
    ├─ 文书名称
    ├─ 案号
    ├─ 审理法院
    ├─ 原告 / 被告 / 第三人
    ├─ 立案时间
    ├─ 审理程序（普通 / 简易）
    ├─ 合议庭 / 独任审理
    └─ 是否公开开庭
   
   【二】诉讼请求（案件边界定义）
    ├─ 原告诉讼请求（编号列明）
    ├─ 诉讼请求变更情况（如有）
    └─ 反诉请求（如有）
   
   【三】事实与理由（原告叙事）
    ├─ 争议背景
    ├─ 行为经过（时间线）
    ├─ 涉案对象说明（合同 / 技术 / 数据 / 劳动关系等）
    └─ 原告主张理由
   
   【四】被告答辩（对抗性叙事）
    ├─ 对诉讼请求的回应
    ├─ 事实抗辩
    ├─ 法律抗辩
    └─ 程序性抗辩（管辖 / 主体 / 时效等）
   
   【五】证据交换与质证（司法证明启动）
    ├─ 原告提交证据
    ├─ 被告提交证据
    ├─ 证据质证情况
    └─ 法院对证据的采信意见
   
   【六】法院认定事实（官方事实版本）
    ├─ 法院确认的基础事实
    ├─ 对关键争议事实的认定
    └─ 排除或不采信的事实说明（如有）
   
   【七】本院认为（第一次法律评价）
    ├─ 争议焦点归纳
    ├─ 法律规范适用
    ├─ 事实与法律的对应分析
    └─ 对各争议焦点的裁判结论
   
   【八】判决结果（实体裁判）
    ├─ 对各诉讼请求的处理
    ├─ 反诉处理（如有）
    ├─ 诉讼费用承担
    └─ 判决生效与上诉权告知
    可插拔模块（一审）
   【A】管辖与程序异议模块
    ├─ 管辖权异议
    ├─ 程序裁定
    └─ 程序变更说明
   
   【B】鉴定 / 技术调查 / 专家意见模块
    ├─ 鉴定申请与准许
    ├─ 鉴定意见
    └─ 法院采信说明
   
   【C】调解 / 和解 / 撤诉模块
    ├─ 调解过程
    ├─ 部分撤诉
    └─ 调解失败说明
   
   【D】行政行为关联模块（混合型一审）
    ├─ 被诉行政行为
    ├─ 行政机关答辩
    └─ 行政证据来源
   ```
2. 让大模型对案例内容进行分块
   - 大模型将原文内容切割，归类到不同的模块中
3. 对每个分块内容进行抽取
   - 对不同模块的内容做不同的知识抽取
   - 同文件不同实体合并规则（依据属性“别称”...）
